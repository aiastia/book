## 一、启动方式

> 推荐用 **`./dev.sh`** 一键启动（自动建 venv / node_modules、后台运行、日志落盘）。

### 方式 A：一键脚本（推荐）

```bash
chmod +x dev.sh        # 首次赋权（仅一次）

./dev.sh               # 同时启动前后端（无参数等同 all）
./dev.sh backend       # 只启后端
./dev.sh frontend      # 只启前端
./dev.sh stop          # 停止所有
./dev.sh status        # 查看运行状态
./dev.sh docker        # 用 Docker Compose 启动
```

启动后：
- 前端：http://localhost:3000
- 后端：http://localhost:8000  （Swagger 文档 http://localhost:8000/docs）
- 日志：`.dev-logs/backend.log`、`.dev-logs/frontend.log`

### 方式 B：手动启动

#### 后端（FastAPI）
```bash
cd backend
python3 -m venv venv            # 首次
source venv/bin/activate
pip install -r requirements.txt # 首次
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### 前端（Nuxt 3）
```bash
cd frontend
npm install        # 首次
npm run dev        # 默认 http://localhost:3000
```

### 方式 C：Docker Compose（生产/隔离环境）

```bash
docker compose up -d --build    # 构建并启动
docker compose logs -f          # 实时日志
docker compose down             # 停止并移除
```
- 前端容器 `mumu-frontend` :3000
- 后端容器 `mumu-backend` :8000，SQLite 数据卷映射到 `./backend/data`（持久化）

### 联调配置
前端 `nuxt.config.ts` 已配置：
```ts
runtimeConfig: {
  apiBase: process.env.NUXT_API_BASE || 'http://localhost:8000',
  public: { apiBase: process.env.NUXT_PUBLIC_API_BASE || 'http://localhost:8000' }
}
```
**默认就能联调**，无需额外配置。如需改地址，用环境变量：
```bash
NUXT_PUBLIC_API_BASE=http://你的后端地址:8000 ./dev.sh frontend
```

## 二、对接架构

### 核心设计：项目级接口 + 前端项目上下文

前端采用 **项目上下文**（`composables/useProject.ts`）管理当前选中的书籍：
- `currentProjectId` 存 localStorage（持久化）+ URL query `?pid=123`（可分享）
- 所有项目内页面（dashboard/chapters/outline/...）通过 `composables/useProjectApi.ts` 调用项目级接口

```
前端页面 → useProjectApi → /api/projects/{pid}/xxx
                ↑
            useProject（提供 pid）
```

### 文件分层

| 层 | 文件 | 职责 |
|---|---|---|
| API 封装 | `composables/useApi.ts` | 统一 GET/POST/PUT/DELETE + token 注入 |
| 项目上下文 | `composables/useProject.ts` | 跟踪 currentProjectId（localStorage + URL） |
| 项目接口 | `composables/useProjectApi.ts` | 所有 `/api/projects/{pid}/...` 调用封装 |
| 鉴权 | `composables/useAuth.ts` | 登录态管理 |

## 七、鉴权策略

- 前端：登录后 token 存 localStorage（`moyu_token`），`useApi.ts` 自动注入 `Authorization: Bearer <token>`
- 后端 `projects.py`：所有接口需 `Depends(get_current_user)` 验证 token
- 后端 `compat.py`（开发友好）：无 token 时回退到数据库第一个用户（**仅开发联调，生产需移除**）
