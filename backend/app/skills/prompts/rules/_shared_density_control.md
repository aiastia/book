职业背景——渗进行为，不挂在比喻上
- 古籍修复者的思维通过动作体现：她判断裂纹的方式、观察纹理的习惯——比"像修复古籍"的明喻有力
- 同一职业比喻来源，整章最多一次。禁止连用四次同类比喻

「不是A，是B」——禁止使用，唯一例外
- 禁止「不是XX，是XX」「不是XX。是XX。」句式。全部改为直接陈述。
- 坏例：「不是矿道，是裂隙」「不是岩石，是胶质」「不是消失，是变形」
- 改法：直接写「矿道裂成了一条向下的裂隙」「岩面覆盖着半透明胶质」「轮廓正在变形」
- 唯一例外：如果"不是A，是B"是角色内心独白的一部分，且是角色自己在纠正自己的判断，允许用 1 次。其余情况均为 0 次。

隐喻密度——2个意象用透，不要10个意象各用一次
- 每章核心隐喻控制在 2~3 个。如果本章用「血/骨/脉搏」建立了基调，后续所有超自然描写必须复用这些意象
- 禁止每段引入一个新概念。已经用了"心跳""锁链""共振"，就不要再引入"琴弦""校准""信标""坐标"
- 每 500 字最多引入 1 个全新的隐喻/设定术语。超出后用已有意象变体表达

概念密度——同一千字内，已出场设定择优复用，勿全量引用
- 禁止在千字内密集引用全部已有设定。挑最重要的 2~3 个延伸，其余留白
- 示例：已出场「倒悬心脏」「第七竖井」「矮人碑文」「墨水孢子」「容照微日记」——这一段只深入展开其中 1~2 个，其他不提
- 读者压力来自信息过载，不是信息不足

一句话内禁止连续堆砌三种及以上修辞
- 坏例：「不是A，是B。像……节奏慢了半拍。」——一句内否定+明喻+节奏评价，三重修辞堆叠
- 一种修辞结束后，用纯粹的动作或叙述句隔开，再过渡到下一种修辞。中间至少隔一句"平淡句"

系统UI辅助原则——系统是触发器，不是主角
- 系统面板、提示音、坐标、数据流、光效、字体、面板动画等 UI 元素属于辅助信息，不得成为描写主体
- 每次出现控制在 1~2 句内，立即回到人物动作/心理/环境反馈/剧情推进
- 禁止连续描写颜色、光效、箭头、数字、字体、音效质感等 UI 细节超过 2 句
- 坏例：「红色箭头。坐标。距离28米。深度。矿层厚度2.3米。全部标注成跳动的数字。」→ 这是 HUD 介绍，不是小说
- 正确：「一道猩红坐标钉进她的视野。目标：裴昭珩。」→ 说完立刻写她的反应
- 系统提示音同理——一句写内容（"叛道者协议——判定非法变量"），下一句立刻切到"她的右手已经不属于自己"

正文重心——每一段回答三个问题
- 每一段写完后，心里检查：①发生了什么？②角色因此做了什么？③局势因角色的行为发生了什么变化？
- 禁止把大量篇幅用于描写系统UI、光效、特效、字体、面板动画、数据流、提示音质感，除非它们本身就是剧情冲突的载体
- 镜头的默认位置：角色 → 失控 → 冲突 → 选择 → 后果。系统永远只是触发器，人物才负责承载
