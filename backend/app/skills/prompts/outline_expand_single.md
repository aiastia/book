<commercial_design_guide priority="P0">
【 商业爽点规划指导 - 每章必须预埋】

规划每一章时，必须为后续正文写作预留明确的爽点发力空间。每章至少体现以下一项：

1. 信息差种子
- 标注本章存在哪组信息不对称：读者知道但某角色不知道 / 主角知道但对手不知道 / 某角色知道但主角不知道
- 明确这个信息差在本章中是被"建立"还是被"推进"
- 如果本章是信息差建立章：读者获得了什么配角不知道的信息？
- 如果本章是信息差推进章：谁接近了真相？谁主动利用了信息差？

2. 震惊/打脸场景预埋
- 如果本章规划了爽点释放，标注震惊层级：点震惊（单人）/ 网震惊（多人不同反应）/ 深度震惊（路人→同行→权威递进）
- 规划围观者分层：吃瓜群众什么反应？同行/内行什么反应？权威/高层什么反应？反派阵营什么反应？
- 每个震惊者的反应必须有差异化——不是所有人都"目瞪口呆"

3. 情绪拉扯节奏
- 标注本章的情绪曲线：从什么情绪出发→经历什么触发→转折到什么情绪→结尾残留什么
- 如果本章是小高潮章（rhythm_tag=小高潮），必须包含一次"以为要输了/不行了→峰回路转"的微拉扯
- 拉扯不需要大起大落——主角差点暴露/差点失败/差点被发现→关键时刻翻盘即可

4. 主角逼格锚点
- 本章主角的逼格通过什么行为展示？（面对挑衅的反应 / 赢了之后的姿态 / 一个让人记住的习惯动作）
- 禁止在规划中写"主角很强""主角装到了"——必须描述具体的行为场景
- 面对挑衅：风轻云淡 >> 暴怒反击；赢了之后：深藏功与名 >> 到处炫耀

上述规划内容填入 shuang_design 字段（结构化对象，见输出格式）。

<constraints>
【 内容边界约束 - 必须严格遵守】
 只能展开当前大纲节点的内容
 深化当前大纲，而非跨越到后续

 不要让剧情快速推进

【展开原则】
 将单一事件拆解为多个细节丰富的章节
 深入挖掘情感、心理、对话和角色间的细微互动
 每章是当前大纲内容的不同侧面或阶段
 环境细节仅作为角色即时感知的载体出现，不做独立的环境描写段落。每章环境描写总量控制在叙述篇幅的15%以内

【 节奏约束 - 每章必须包含】
1. 一个角色正在做的具体的事（或正在接收的具体信息）
2. 一个阻碍这件事的因素（人、环境、信息缺失、规则、另一个角色的反对）
3. 角色对阻碍做出的反应（行动或选择，包括选择暂时不做反应）
4. 做完反应后，有一个具体的变化——手里多了东西、知道了坏消息、和某人的距离变了、一个选择被做出来了
5. 一个钩子——发生在上述变化之后，把读者拽向下一章

【 商业爽点约束】
- 每章 shuang_design 必须填写完整（5个子字段缺一不可）
- 至少每隔一章存在一组信息差（info_asymmetry 不为空）
- 小高潮章（rhythm_tag=小高潮）的 shock_level 必须至少为"网震惊"
- 同一大纲节点内的章节，spectator_layers 不能每章相同——不同章聚焦不同层级的围观者反应
- protagonist_style 必须用具体行为描述，禁止写"主角装到了""主角很强"等抽象概括

【 三章节奏规则】
如果{target_chapter_count}≥3：第3章（sub_index=3）的rhythm_tag必须填"小高潮"，且该章的hook强度必须高于前两章——不是"产生疑问"，而是"产生紧迫感"。小高潮满足以下至少一项：
- 一场对抗的结果揭晓（战斗/博弈/试探的胜负）
- 一个秘密被揭开（角色获知了之前不知道的关键信息）
- 一段关系的性质发生变化（同盟破裂/敌人被迫合作/暧昧变成明牌）
- 一个角色做出了不可逆的选择

【 情节点优先级与数量约束】
- 每章 key_events 总数 3-4 个，不超过 4 个
- 每个事件前标注优先级：【重点】/【一般】/【弱】
- 【重点】每章不超过 2 个，为核心事件（本章入口 + 核心转折），必须完成
- 【一般】重要但有余量时再写
- 【弱】锦上添花，字数不够可省略
- 排序原则：重点项先行，一般/弱项靠后

【 相邻章节差异化约束（防止重复）】
 每章有独特的开场方式（不同场景、时间点、角色状态）
 每章有独特的钩子（不同悬念、转折、情感收尾类型）
 key_events在相邻章节间绝不重叠
 plot_summary描述该章独特内容，不与其他章雷同
 同一事件的不同阶段要明确区分"前、中、后"

【章节间要求】
 衔接自然流畅（每章从不同起点开始）
 剧情递进合理（但不超出当前大纲边界）
 节奏张弛有度——'起'章收紧，'承'章升温，'小高潮'章引爆
 每章有明确且独特的叙事价值
 最后一章结束时恰好完成当前大纲内容
 关键事件无重叠：检查相邻章节key_events

【禁止事项】
 输出非JSON格式
 剧情越界到后续大纲
 相邻章节内容重复
 关键事件雷同
 plot_summary前三分之一以环境描写或背景介绍开头
 在hook中使用总结句（包括"他不知道的是""等待他的将是""这一切都变了"等）
 拆分后的章节缺乏阻碍-反应-变化链条
</constraints>
