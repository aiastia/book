【第零步：题材锁定】（最高优先级）
- 你生成的所有内容必须严格属于「{genre}」题材，不得跨题材。
- 命名风格必须与题材匹配：东方题材（仙侠/玄幻/武侠/历史等）用中式命名；西方题材（西方奇幻/魔幻等）用译制风格西式命名；科幻/末世按题材气质定。
- 如果是东方题材（仙侠/玄幻/历史等），用"修真体系/宗门/灵脉"等设定；如果是西方题材（奇幻/魔幻），用"魔法体系/骑士团/信仰/行会"等设定；不要混用东西方设定框架。
- 严禁现代科技/科幻元素乱入到非科技题材中（古典/架空/奇幻/玄幻/仙侠/武侠/历史类禁止科学家/机甲/人工智能等）。

【第一步：判断世界观结构类型】（内部判断，不输出，但决定你使用哪组条目类别）

■ 单世界型（仙侠/玄幻/都市/历史/奇幻/科幻单星球等）
  使用类别组A：地理 / 历史 / 种族 / 势力组织 / 力量体系 / 文化 / 经济 / 军事 / 宗教信仰 / 风俗 / 其他
  （注意："力量体系"和"势力组织"要按题材填——仙侠填修炼境界/宗门，西方奇幻填魔法体系/骑士团/神殿，不要套错框架）

■ 多位面穿梭型（快穿/无限流/诸天万界/位面交易/综穿等）
  使用类别组B：系统空间 / 位面分类 / 常见位面模板 / 穿越者组织 / 任务机制 / 积分兑换 / 规则边界 / 位面意志 / 身份生成 / 异常事件 / 其他

■ 表里双世界型（现实+异界/平行世界等）
  使用类别组A和B的混合，侧重两世界对照：表世界地理 / 里世界地理 / 边界与通道 / 双世界交互规则 / 两边势力对照 / 其他

■ 轮回重置型（重生循环/时间回溯）
  使用类别组C：世界基线 / 关键节点 / 循环变量 / 记忆继承 / 蝴蝶效应案例 / 终局路径 / 其他

【第二步：按类型生成条目】

每个条目 200-400 字，内容具体、有细节、有画面感。

⚠️ 重要原则：
- 快穿题材下，不要生成具体位面的地理/历史/种族——那是后续为每个位面单独做的事。
  此时应专注生成"元层面"的设定：系统空间长什么样、位面如何分类、任务怎么运作、积分换什么。
- 如果核心世界观描述的确实是单世界（如修真大陆/奇幻大陆），则使用类别组A正常生成实体条目。
- 如果拿不准类型，默认使用类别组A。
- 条目内容要与上方提供的【世界观】四维度保持一致，不要自相矛盾。

【命名硬性规则】
- 禁用字：烬、殇、弑、戮、煞、戾（已被网文用到彻底失去吸引力）
- 避免空洞堆砌词：帝/皇/王/神/圣/尊（太直白的"强"反而弱）、逆天/无敌/至尊（小白文既视感）
- 好的名称用具体意象代替抽象滥用字。坏例："灰烬王庭""殇之谷"。好例："锈蚀矿脉""缄默之森""巨人之踵""霜棘隘口"
- 东方题材另禁姓氏：沈、苏、谢（网文烂大街三姓）
