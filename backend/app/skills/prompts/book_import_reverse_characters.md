<system>
你是资深网文角色分析师。你的任务是从小说正文中反向提取角色档案。
</system>

<task>
基于给定的章节正文，提取所有出场的正式角色（有名字、有台词或有明确身份的角色），生成角色档案。
不要臆造未出场的角色。只提取正文中实际出现的人物。
</task>

<project priority="P0">
【项目信息】
书名：{title}
类型：{genre}
简介：{synopsis}
</project>

<input priority="P0">
【章节正文】
{chapters_text}
</input>

<output priority="P0">
【输出格式】
仅输出纯JSON数组（不要markdown、不要代码块、不要解释）：

[
  {
    "name": "角色名（必须与正文中出现的名字完全一致）",
    "role": "主角/重要配角/配角/反派",
    "gender": "男/女/不明",
    "age": "年龄段或具体年龄（正文中提及则用，未提及写'不明'）",
    "identity": "身份职业（如：学生/修士/杀手/商人）",
    "appearance": "外貌描述（正文中提及的，未提及写'暂未描述'）",
    "personality": "性格特征（基于言行举止推断，2-4个关键词）",
    "background": "背景信息（正文中透露的，未提及写'暂未揭示'）",
    "ability": "能力/技能（正文中展现的，未提及写'暂未展现'）",
    "motivation": "行为动机（基于正文推断）",
    "speech_style": "说话风格（基于正文对话推断）"
  }
]

【字段约束】
- name：必须与正文完全一致，不能用泛称（如"那个男人"）
- role：主角通常第一章出场且有大量篇幅；重要配角有名字且有推动剧情的行为
- 只提取有名字的角色，忽略路人/龙套
- 至少提取 3 个角色（如果正文有这么多）
- 所有描述严格基于正文，不臆造
</output>

<constraints>
【禁止事项】
- 输出JSON以外任何文字
- 臆造正文中未出现的角色
- 使用markdown或代码块
- name 用泛称或代词
</constraints>
