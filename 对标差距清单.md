## 二、AI 调用鲁棒性（最致命短板，P0 必先修）

### 2.1 JSON 清洗
- **原项目** `json_helper.py`（680 行）4 层修复：
  1. 上下文感知的中文标点→ASCII 转换（区分字符串内外）
  2. 字符串内裸换行/制表符转义、中文引号转义
  3. 未转义 ASCII 引号智能修复（看后续字符判断）
  4. 基于 `json.JSONDecodeError.pos` 精确错误位置迭代修复（最多 50 次）
  5. json5 兜底解析
- **墨语** `ai_client.py` chat_json（30 行）：只去 markdown 包裹 + 简单截取 `{...}` / `[...]`
- **差距**：中文引号/未转义引号/裸换行/无效转义全部无法处理 → AI 返回稍不规范就解析失败

### 2.2 重试机制
- **原项目** `call_with_json_retry`（3 次重试 + hint 强化 + expected_type 校验 + AICallMetrics 埋点）
- **墨语**：完全无重试，失败直接返回 error

### 2.3 多 Provider
- **原项目**：OpenAI / Anthropic / Gemini 三套 client + provider + 适配器（xiaomi_mimo）
- **墨语**：仅 OpenAI 兼容

## 三、核心数据模型字段差距

### 3.1 Project（P0/P1 缺失字段）
| 缺失字段 | 影响 | 优先级 |
|---|---|---|
| `outline_mode`（one-to-one/one-to-many） | 项目级大纲生成模式开关缺失 | P0 |
| `wizard_status` / `wizard_step` | 创作向导无法断点续写 | P0 |
| `theme` | 创作元信息 | P1 |
| `character_count` | 角色数量规划 | P1 |
| `cover_prompt` / `cover_status` / `cover_error` / `cover_updated_at` | 封面生成状态机 | P1 |

命名差异（需映射）：description↔synopsis、target_words↔target_word_count、current_words↔current_word_count、narrative_perspective↔narrative_pov、cover_image_url↔cover_url

### 3.2 Chapter
| 缺失字段 | 影响 | 优先级 |
|---|---|---|
| `sub_index`（大纲下子章节序号） | 1:N 模式无法定位子章节 | P0 |
| `expansion_plan`（结构化） | 原是 Text JSON，墨语是 JSON，可兼容 | — |

### 3.3 Character
| 缺失字段 | 影响 | 优先级 |
|---|---|---|
| `status_changed_chapter` / `state_updated_chapter` | 角色状态变更无章节溯源 | P1 |
| `main_career_id` / `main_career_stage` / `sub_careers` | 副职业体系缺失 | P1 |
| `avatar_url` | 头像缺失 | P2 |
| 墨语 `mental_state` 是 String(50)，原 `current_state` 是 Text | 类型受限 | P1 |

架构差异：原项目角色+组织合一（`is_organization` 标志），墨语拆成 Character + Organization 两表。

### 3.4 Foreshadow（伏笔，差距最大）
原项目 40+ 列完整伏笔管理，墨语仅 14 列。缺失关键：
| 缺失字段 | 影响 | 优先级 |
|---|---|---|
| `is_long_term` + status 的 `partially_resolved` 枚举 | 长线伏笔/部分回收态 | P0 |
| `auto_remind` / `remind_before_chapters` / `include_in_context` | 章节生成伏笔提醒 | P0 |
| `hint_text` / `resolution_text` | 原文摘录/回收文本 | P1 |
| `importance` / `strength` / `subtlety` / `urgency` | 强度/隐蔽度评分 + 紧急度算法 | P1 |
| `related_characters` / `related_foreshadow_ids` | 伏笔链 + 关联角色 | P1 |
| `category`（identity/mystery/item/...） | 分类体系（墨语用 foreshadow_type 中文） | P1 |
| `notes` / `resolution_notes` | 创作备注 | P2 |
| `planted_at` / `resolved_at` | 时间戳 | P2 |
| `source_memory_id` / `source_analysis_id` / 三个 `*_chapter_id` 外键 | 分析回溯 | P2 |

### 3.5 World
架构差异：原项目世界观 4 字段内嵌 Project，墨语独立 WorldSetting 表（表达能力更强，迁移时原 4 字段→4 条记录）。

### 3.6 Outline
命名差异：content↔summary、order_index↔sort_order。墨语额外有 scenes/characters/key_points/emotion/goal（更显式）。

## 四、功能模块差距

### 4.1 灵感模式（inspiration）
| 能力 | 原项目 | 墨语 | 差距 |
|---|---|---|---|
| 步骤 | 4 步 AI + perspective + outline_mode + confirm | 仅 4 步 AI 选择 | ❌ 缺 perspective/outline_mode/confirm |
| 生成方式 | 流式累积 | 一次性 | ❌ |
| 温度策略 | 递减（title 0.8 → genre 0.45） | 无 | ❌ |
| 重试 | 3 次 + prompt 强化 | 无 | ❌ |
| 格式校验 | validate_options_response（3-10 选项等） | 无 | ❌ |
| refine-options | ✅ 用户反馈重新生成 | ❌ 无 | ❌ |
| 失败降级 | 返回"让AI重新生成/自己输入" | 报错 | ❌ |
| 前端缓存 | localStorage 24h 对话恢复 | 无 | ❌ |

### 4.2 项目创建向导（wizard_stream）
- **原项目**：4 个 SSE 生成器串联（world_building → career_system → characters → outline），1797 行，有 WizardProgressTracker 进度事件、分批角色（每批5个）、数量校验、幻觉关系清理、角色→职业→组织→关系五阶段事务、wizard_step 状态机。
- **墨语**：完全没有向导，世界/角色/大纲各自独立一次性接口，无串联、无进度、无重试。
- **差距**：❌ 完全缺失向导式流程（这是原项目核心入口）

### 4.3 章节生成
- 框架相似（两种模式 + 流式 + 后处理），但**上下文信息密度严重不足**：
  - ❌ 无 expansion_plan 结构化解析（plot_summary/key_events/character_focus）
  - ❌ 角色无关系网络/组织/职业阶段关联查询
  - ❌ 记忆无向量检索（墨语是 importance 排序）
  - ❌ 无上一章"关键事件"提取（只有末尾文本）

### 4.4 剧情分析（plot_analysis）
- ❌ 无重试 + 指数退避 + 回调
- ❌ 记忆提取维度少（原 6 类 + keyword 定位，墨语 2 类无定位）
- ❌ 无已有伏笔分层注入
- ❌ 无番茄平台评分维度

### 4.5 完全缺失的功能模块
| 功能 | 状态 |
|---|---|
| 写作风格管理 writing_styles | ❌ 仅 project.writing_style 一个 JSON 字段 |
| 角色关系图谱 relationships | ❌ 无 |
| 职业体系 careers（落库） | ⚠️ 有生成接口但不落库 |
| 封面真实出图 | ❌ 只有提示词生成 |
| 后台批量任务 tasks | ❌ 完全无 |
| MCP 自动工具加载 | ⚠️ 有接口但 AIClient 不支持 tools |
| 提示词工坊 prompt_workshop | ❌ 完全无 |
| 用户管理后台 / 公告 / 更新日志 | ❌ |
| OAuth 登录 | ❌ |
| 记忆系统（向量检索） | ⚠️ 有模型但无向量检索/独立 API |
